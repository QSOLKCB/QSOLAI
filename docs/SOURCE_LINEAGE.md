# Source and conceptual lineage

QSOLAI v0.1.0 was written as a clean Python standard-library implementation. Reference code was inspected for architecture and tests, but no literal source fragment was adapted.

| Reference | Conceptual influence | Boundary |
|---|---|---|
| [QEC](https://github.com/QSOLKCB/QEC) | canonical identities, replay-safe receipts, observation traces, adapter-not-authority rules, claim scopes, child-first validation | QSOLAI does not copy QEC receipt modules or claim quantum-error-correction functionality. QEC attribution is preserved under CC BY 4.0. |
| [RES=RAG](https://github.com/QSOLKCB/res-rag) | grounding/substrate versus generation; tool-first functional interaction | No consciousness implementation or empirical validation claim. |
| [SONIFICATION](https://github.com/QSOLKCB/SONIFICATION) | integer-only canonical JSON, domain hashes, source-bundle identity, self-hash exclusion, committed event identity, safe output directories | Clean Python reimplementation; no ETQ or sonification semantics. |
| [SPECTRAL](https://github.com/QSOLKCB/SPECTRAL) | strict/captured replay boundaries, explicit registries, deterministic ZIPs, identity/presentation separation, known-answer tests and offline audits | No browser, DSP or WAV implementation. |
| [Ternary Drift](https://github.com/QSOLKCB/GAMES/tree/main/TERNARYDRIFT) | authoritative/non-authoritative state, isolated streams, exact replay and hard size verification | No game code or Win32 runtime. |

The resulting architecture belongs to QSOLAI and is distributed under MPL-2.0. Future literal adaptations must identify the exact file and preserve every required notice before merge.
